1. Entry Conditions

开工前先锁住边界

Scope Lock

只做 Replay A 的可运行闭环，外加 Replay B 所需的语义位点。真实 Commit 执行仍然不进入本轮。

Truth Shape

底层必须是 append-only event log，上层是 materialized view。不得直接把一份内存对象树当唯一真相实现。

Proof Goal

本轮证明的是授权、验证、时间语义、冲突语义、低打扰选择器，而不是“系统会很多能力”。

Start Rule: 如果某项工作不能直接改善 replay load / event provenance / fact semantics / grant guard / verification gate / selector attention / capsule export 之一，就不应进入 M0/M1。

2. Delivery Tasks

把 W1-W7 压成 12 个可执行任务

T01

Schema Gateway

Maps to W1

建立一个统一入口读取 schemas/json-schema 与 schemas/examples，确保运行时消费的是同一套 contract，而不是再手写一套平行对象。

Input

现有 JSON Schema 与 example payload

Output

统一加载器和解析失败模型

Done

无业务代码时即可装载 Replay A timeline

Fail Signal

装载时默认补字段或吞掉 schema 错误

T02

Replay Loader

Maps to W1

把 flow fixture 编译成有因果顺序的 timeline，而不是随手遍历 JSON。timeline 需要保留 event causation、expected state changes 和 assertions 位点。

Input

task-replay.schema.json 与两条 replay

Output

可执行 timeline + step index

Done

Replay A/B 都能被解析成稳定步骤序列

Fail Signal

顺序依赖靠运行时猜测而不是 fixture 显式表达

T03

Event Append Path

Maps to W2

实现 append-only 写入路径，强制 event envelope 持有 event_id / causation / correlation / trace / recorded_at，并且做按 event_id 去重。

Input

EventEnvelope schema

Output

写入接口、dedup guard、append test

Done

重复 append 不生成重复事件

Fail Signal

任何业务层可直接改写历史事件

T04

Projector Bootstrap

Maps to W2/W3

建立从事件流到物化视图的单向 projector，最少投影 TaskNode、Fact、Grant、Receipt、Verification 五种视图。

Input

append-only event log

Output

deterministic projector + rebuild path

Done

删除缓存后仍能完整重建当前视图

Fail Signal

重建必须依赖隐藏内存状态

T05

Task Transition Guards

Maps to W4

把 created -> ready -> running -> verifying -> completed 等节点迁移守卫实现成显式规则，不允许靠调用顺序隐式推进。

Input

Task state machine contract

Output

transition validator

Done

无 verification 不能进 terminal

Fail Signal

execution path 可直接写 completed

T06

Fact Time Semantics

Maps to W4

把事实从静态值改成带时间和来源的声明：最少落实 fact_type / provenance / observed_at / verified_at / valid_until / ttl / attestation_level。

Input

Fact schema 与 replay assertions

Output

freshness evaluator

Done

过期事实自动降为 stale

Fail Signal

verified fact 被当永久真相

T07

Fact Conflict Semantics

Maps to W4

落实 conflict_set / supersedes 语义，让冲突事实先并存，再经裁决事件收敛，而不是谁后写谁覆盖。

Input

candidate 与 verified fact events

Output

conflict-aware view logic

Done

冲突事实在未裁决前稳定并存

Fail Signal

last write wins 掩盖冲突

T08

Grant Guard

Maps to W5

建立最小授权守卫，检查 who / what / where / when / budget / why 是否匹配 action，不允许“先跑起来再补 grant”。

Input

CapabilityGrant schema

Output

active grant evaluator

Done

无 active grant dispatch 被拒绝

Fail Signal

低风险动作默认绕过授权

T09

Structured Dispatch + Receipt

Maps to W5

只做本轮需要的低风险结构化动作，并强制每次执行都落一条 receipt。dispatch 成功不等于任务完成。

Input

ActionSpec 与 local execution adapter

Output

dispatch path + receipt recorder

Done

每次副作用都有可追溯 receipt

Fail Signal

执行成功却无 receipt

T10

Verification Gate

Maps to W5

把 terminal eligibility 收到验证器里，验证器只看任务目标、postcondition、receipt、evidence，而不看“调用方是不是觉得已经完成”。

Input

VerificationResult schema

Output

verification evaluator

Done

无 verification terminal transition 被拒绝

Fail Signal

任何 adapter 可直接宣告 done

T11

Selector v0 with Attention Budget

Maps to W6

选择器必须先做硬约束过滤，再按 freshness、health、verification strength、risk、attention、cost 评分。默认路径顺序只能当 tie-breaker，不能写成不可挑战的全序。

Input

candidate actions + policy context

Output

ranked next-step result

Done

Replay A 在不过度 ask 的情况下完成

Fail Signal

attention budget 未进入 decision rationale

T12

Minimal Capsule Export

Maps to W7

只做 resume / handoff 最小 capsule 导出，且必须区分 hard constraints 与 soft notes。capsule 只引用真相切片，不成为新真相副本。

Input

materialized state slice

Output

capsule serializer + validator

Done

导出后可校验 hard constraints 与 freshness markers

Fail Signal

capsule 被用来替代共享状态面

3. Pull Order

任务顺序按因果链走，不按模块热情走

WaveTasksWhy This OrderShould Not Happen Yet
Wave 0T01, T02先锁 contract 与 replay 输入，不然后续所有运行时都会漂移自定义平行对象模型
Wave 1T03, T04先有事件来源和 projector，后面才有真实状态语义直接堆 service 内存状态
Wave 2T05, T06, T07先把 Task / Fact 语义钉死，再接执行和选择先做 UI 或广兼容 adapter
Wave 3T08, T09, T10完成 grant -> dispatch -> receipt -> verification 的闭环真实 Commit 执行
Wave 4T11, T12最后再接 selector 和 capsule，避免两者掩盖下层漏洞多 agent handoff / merge review

6. Exit Gate

只有满足这些，才允许走向 M2

Must Be Green
  • T01-T10 全部通过
  • Replay A 可完整回放
  • grant / verification 无绕路
  • Fact 时间与冲突语义已测
Can Still Be Narrow
  • T11 只要能解释 decision rationale
  • T12 只要支持最小 resume / handoff
  • Procedure 仍可停在 draft/trial
Do Not Expand Yet
  • 多 agent orchestration
  • 真实 Commit 执行
  • 全量 compatibility mesh
  • 复杂 memory ranking

M2 Trigger: 只有当 M0/M1 已经证明“没授权不执行、没验证不完成、事实不会静默过期污染、选择器能控制打扰成本”后，才值得进入 Replay B 的真实 Operate / Commit / compensate 闭环。
