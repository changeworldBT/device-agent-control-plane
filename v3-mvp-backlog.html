What This Proves

没授权不执行，没验证不完成，失败能收敛，经验能开始编译，且用户注意力不会被无意义打断消耗掉。

What This Avoids

不在第一版同时做多 agent、全兼容层、全量 memory compiler、广覆盖 task taxonomy 和复杂 pack 发行。

1. Backlog Position

这是实现 companion，不是新的冻结层

Canonical

v3-design-freeze.html 仍然是唯一规范正文。

Derived

本文件把 replay 里的不变量、默认协议和策略面转成实现顺序。

Adjustable

这里的里程碑、切片和优先级都可改；只要不破坏 canonical freeze 的不变量即可。

Backlog Rule: 如果某项工作不能直接提升“授权、验证、恢复、时间语义、事实语义、事件可追溯”中的至少一项，就不应进入首个 MVP。

2. Proof Target

首个 MVP 只需要证明 5 件事

  • 没有 active grant 的动作不会被 dispatch。
  • 没有 verification result 的节点不会进入终态。
  • 事实带时间、来源、冲突位点，不会被静默覆盖。
  • 不可完全回滚的动作会走 compensation 语义，而不是假装 rollback。
  • 单次成功之后，系统能开始积累可验证的 procedure 候选，而不是只积累聊天记录。
Two Replay Targets

Replay A：研究简报链，证明 Observe/Synthesize/Prepare 的闭环。

Replay B：CRM 更新 + 受控外发链，证明 Operate/Commit、approval、verification 和 compensation 的闭环。

3. Explicit Non-Goals

第一阶段明确不做的东西

Not Now

多 agent 调度和 merge review 的完整实现。第一阶段只保留 lease / handoff 语义位点。

Not Now

广覆盖生态兼容。只接 replay 必需的本地能力和最小外部系统适配器。

Not Now

全量 memory compiler。第一阶段只支持事实、失败和 procedure draft/trial 候选。

Not Now

像素 fallback 和脆弱 GUI 自动化。第一阶段优先结构化适配器和本地文件能力。

Not Now

多设备、多身份全展开。只保留 principal / owner / approver 位点，不扩展复杂主体系统。

Not Now

动态风险学习和自动调参。第一阶段只实现组合风险函数的静态版本。

4. Work Packages

最小实现切片

M0

Replay Harness and Validation Spine

先把 schemas/、event flow、example validation、fixture replay runner 打通。没有这层，后面每个模块都会按自己的理解实现对象语义。

  • 消费 [schemas/json-schema](C:/Users/84915/device-agent-doc/schemas/json-schema) 和 [schemas/examples](C:/Users/84915/device-agent-doc/schemas/examples)
  • 建立 append-only event log 的最小记录格式
  • 从事件流物化 TaskNode / Fact / Grant / Receipt / Verification 视图
  • 把两条 replay fixture 跑成可审计 timeline

Exit: 两条 replay 都能被读取、执行为状态推进模拟，并输出一致的 materialized view。

M1

Single-Agent Research Brief Vertical Slice

先做纯 Observe/Synthesize/Prepare 闭环，避免一上来就陷入 commit 风险和 UX 打断。

  • Task / TaskNode 生命周期
  • Fact 的时间、来源、TTL、conflict_set、supersedes
  • Grant v0：只支持低风险 scoped grant
  • Selector v0：硬约束 + 默认评分，不做复杂 learning
  • Verifier v0：artifact existence + citation completeness + freshness checks
  • StateCapsule v0：只支持 handoff / resume 的最小验证

Exit: 研究简报 replay 跑通，且能证明“conflicting facts coexist until verified resolution”。

M2

Controlled Operate / Commit Slice

在 M1 之上引入有限写入和受控提交，验证 Prepare/Commit barrier、approval、rollback/compensate 分流。

  • 组合风险函数 v0：action × target × context × sequence
  • R2 / R3 grant issuance and explicit approval binding
  • Operate adapter：可回读、可回滚的字段写入
  • Commit adapter：不可逆动作的 compensation contract
  • Verifier v1：外部回执 + postcondition + compensation visibility

Exit: CRM / Commit replay 跑通，且“邮件已发出只能补偿不能回滚”的语义被系统正确表达。

M3

Procedure and Capsule Hardening

不是全量 memory compiler，而是让系统开始从重复成功中产生可测的成本下降。

  • Procedure draft/trial lifecycle
  • Capsule validator for hard/soft constraints
  • Fact freshness invalidation and stale downgrade
  • Basic attention-budget accounting inside selector

Exit: 第二次运行同类任务时，selector 能利用 procedure candidate 减少 planner 介入或人工打断。

5. Concrete Build Order

实现顺序按因果链，而不是按模块名

OrderCapabilityWhy FirstReplay Coverage
1Event log + projector没有事件来源就没有可追溯真相面A / B
2TaskNode transitions没有节点语义就无法验证终态A / B
3Grant issuance + lease checks没有授权约束就只是工具编排A / B
4Receipt writer + verifier没有 receipt / verification 就没有“完成”语义A / B
5Fact time/conflict model没有时间语义，长期运行必然被旧事实污染A
6Composite risk scorer没有组合风险，Commit 治理会退回静态标签B
7Compensation path没有补偿语义，外发类动作无法真实收敛B
8Procedure/capsule hardening开始兑现“越用越便宜”的承诺A / B
6. First Tests

先写哪类测试，决定系统会不会走偏

Must-Have Tests
  • 无 grant dispatch 拒绝
  • 无 verification terminal transition 拒绝
  • expired fact auto-downgrade
  • conflict facts coexist until explicit resolution
  • commit without approval rejected
  • irreversible send produces compensate path, not fake rollback
Tests to Delay
  • 多 agent merge conflict matrix
  • 复杂 provider compatibility surface
  • full memory ranking and retrieval tuning
  • GUI pixel fallback robustness
7. MVP Exit

什么叫“第一版已经站住”

Replay A Pass

研究简报链跑通，事实有时间语义，冲突可并存，Prepare 产物可验证完成。

Replay B Pass

CRM 更新和外发链跑通，R2/R3 审批生效，Commit 有 compensation 语义。

Cost Signal Appears

第二次同类任务比第一次更少打断、更少 planner、更多命中已有结构。

Do Not Expand Yet: 如果这三条还没成立，就不要扩多 agent、广兼容、复杂 pack、分布式或大规模策略调优。那只会把问题从“还没跑通核心语义”变成“系统更大但更难诊断”。
